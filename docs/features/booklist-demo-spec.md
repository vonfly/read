# 书架列表 Spec

## 要解决的问题
用户打开 App 后需要看到自己导入的所有书籍，并能快速进入阅读。

## 用户故事
- 作为读者，我可以看到书架上所有书籍的封面、书名、作者和阅读进度，以便快速找到想读的书
- 作为读者，我可以点击任意书籍直接进入阅读器，以便无需额外操作继续上次阅读
- 作为读者，我可以长按书籍弹出操作菜单删除它，以便管理书架

## 验收标准
- [ ] 书架列表从 Room 数据库加载，断网状态下仍可正常显示
- [ ] 每本书显示：封面图（无封面时显示占位图）、书名、作者、阅读进度百分比
- [ ] 书架为空时，显示"还没有书籍，点击右下角导入"的空状态引导页
- [ ] 点击书籍跳转到阅读器页面，传入 bookId
- [ ] 长按书籍弹出确认删除 Dialog，确认后从数据库删除并刷新列表
- [ ] 列表使用 Paging 3 加载，滑动流畅无卡顿

## 不在本期范围内
- 书籍排序和筛选
- 书架分组 / 书单功能
- 搜索书籍
- 云端同步书架
- 批量操作（多选删除）

## 技术约束
- 数据层：`BookEntity` 和 `BookDao` 需要在本期创建
- 分页：使用 Paging 3，参考 `docs/templates/paging.md`
- 导航：完成后需要更新 `AppNavHost.kt`，将 `HomeRoute` 替换为 `BookListScreen`
- 封面图加载：使用 Coil
